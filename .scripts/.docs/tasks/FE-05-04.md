# FE-05-04 — Form Validation and Submission Error Tests

**Epic:** [FE-05 — Testing](../epics/FE-05-testing.md)  
**Status:** `[x]` Done

## Description
Write tests for form components that cover valid submission, field-level validation errors, and server-side error display.

## Acceptance Criteria
- [x] Test: `Form_WhenRequiredFieldEmpty_ShowsValidationError`
- [x] Test: `Form_WhenValidInput_SubmitsAndCallsHandler`
- [x] Test: `Form_WhenServerReturnsError_DisplaysServerError`
- [x] Tests simulate user interaction with `userEvent.type()` and `userEvent.click()`.
- [x] Validation error messages are asserted by their accessible label/role, not CSS class.
- [x] Test: field with invalid format shows the Zod schema error message.
- [x] Test names follow `ComponentOrHook_Condition_ExpectedResult` convention.

## Technical Notes
- Use `screen.getByRole('alert')` or `screen.getByText(errorMessage)` to assert error messages.
- Mock the submit handler to verify it was called with the correct typed data.

## Definition of Done
All form tests pass; error messages are visible to screen readers (asserted via accessible queries).
