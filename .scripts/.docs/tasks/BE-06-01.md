# BE-06-01 — Set Up Test Projects (Unit + Integration)

**Epic:** [BE-06 — Quality and Standards](../epics/BE-06-quality-standards.md)  
**Status:** `[x]` Done

## Description
Create the test project structure with separate projects for unit tests and integration tests, both using xUnit.

## Acceptance Criteria
- [x] `tests/Domain.UnitTests/` project exists and references only `Domain`.
- [x] `tests/Application.UnitTests/` project exists and references `Application` + mocking library.
- [x] `tests/Api.IntegrationTests/` project exists and references `Api`.
- [x] All test projects use xUnit and `FluentAssertions`.
- [x] `dotnet test` runs all test projects from the solution root.
- [x] Test projects do not reference each other.

## Technical Notes
- Use `dotnet new xunit` for test projects.
- Add `FluentAssertions` and `Moq` (or `NSubstitute`) to unit test projects.
- Add `Microsoft.AspNetCore.Mvc.Testing` to integration test project.

## Definition of Done
All test projects compile and `dotnet test` exits 0 with the initial placeholder tests.
