# FE-01-04 — Configure Vitest + React Testing Library

**Epic:** [FE-01 — Project Setup](../epics/FE-01-project-setup.md)  
**Status:** `[x]` Done

## Description
Install and configure Vitest as the test runner with React Testing Library for component tests and jsdom as the test environment.

## Acceptance Criteria
- [x] Vitest is installed and configured in `vite.config.ts` (or `vitest.config.ts`).
- [x] `@testing-library/react`, `@testing-library/user-event`, and `@testing-library/jest-dom` are installed.
- [x] `jsdom` is configured as the test environment.
- [x] `setupTests.ts` imports `@testing-library/jest-dom` matchers.
- [x] `npm run test` runs all tests and exits 0 on the initial sample test.
- [x] `npm run test:coverage` generates a coverage report.

## Technical Notes
- Configure `globals: true` in Vitest to avoid importing `describe`/`it`/`expect` in every file.
- Add `setupFiles: ['./src/setupTests.ts']` to the Vitest config.

## Definition of Done
A sample component test renders a `<div>` and asserts on text content; all commands exit 0.
