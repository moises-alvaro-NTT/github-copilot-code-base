# FE-05-01 — Unit Tests for Hooks and Utilities

**Epic:** [FE-05 — Testing](../epics/FE-05-testing.md)  
**Status:** `[x]` Done

## Description
Write unit tests for all shared hooks (`useSessionStore`, custom hooks) and utility functions (`cn`, Zod schemas).

## Acceptance Criteria
- [x] `cn()` utility is covered by tests: merge conflict, falsy input, empty input.
- [x] Zustand store(s) have tests for initial state and each state-mutating action.
- [x] Custom data hooks are tested with `renderHook` and mocked query client.
- [x] Zod schemas have tests for valid input (passes) and each invalid field (fails with correct message).
- [x] Test names follow `ComponentOrHook_Condition_ExpectedResult` convention.
- [x] All test code is in English.

## Technical Notes
- Use `renderHook` from `@testing-library/react` for custom hook tests.
- Wrap hook tests in a `QueryClientWrapper` when testing TanStack Query hooks.

## Definition of Done
All utility and hook unit tests pass; no test touches the DOM or network.
