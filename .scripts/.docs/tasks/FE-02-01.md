# FE-02-01 — Add React Router and Define Route Structure

**Epic:** [FE-02 — Core Libraries Integration](../epics/FE-02-core-libraries.md)  
**Status:** `[x]` Done

## Description
Install React Router and define the top-level route tree with placeholder pages for each known route.

## Acceptance Criteria
- [x] `react-router-dom` v6+ is installed.
- [x] A `Router` component wraps the app in `main.tsx`.
- [x] At least two routes are defined (e.g., `/` and `/renegotiations`).
- [x] A shared `Layout` component with navigation renders for all routes.
- [x] A `NotFound` page renders for unmatched paths.
- [x] Route definitions are in a single `routes.tsx` file (or `app/router.tsx`).
- [x] Navigation links use `<Link>` — not `<a href>`.

## Technical Notes
- Use `createBrowserRouter` + `RouterProvider` (React Router v6.4+ data API).
- Lazy-load feature pages with `React.lazy` + `Suspense` to reduce initial bundle.

## Definition of Done
Navigation between two routes works; 404 page renders for unknown paths; no `<a href>` page reloads.
