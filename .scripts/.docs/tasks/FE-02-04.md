# FE-02-04 — Add Zustand for Shared State (When Needed)

**Epic:** [FE-02 — Core Libraries Integration](../epics/FE-02-core-libraries.md)  
**Status:** `[x]` Done

## Description
Install Zustand and create a minimal store as a pattern baseline. Only add stores for state that genuinely needs cross-feature sharing (e.g., current user session, notification queue). Do not over-use.

## Acceptance Criteria
- [x] `zustand` is installed.
- [x] At least one store (e.g., `useSessionStore`) is created in `lib/stores/`.
- [x] Store uses `create<StateType>()` with a typed state interface.
- [x] Store is used in at least one component and state updates re-render correctly.
- [x] Store does not duplicate state already managed by TanStack Query (server state).
- [x] Store file is covered by a unit test asserting initial state and an action.

## Technical Notes
- Keep stores small and focused; do not create a global monolithic store.
- Prefer `useQuery` for server data; Zustand for pure client-side UI state.

## Definition of Done
Sample store compiles, updates state correctly, and is covered by a unit test.
