# FE-04-04 — Handle Loading, Empty, and Error States Consistently

**Epic:** [FE-04 — Feature Modules](../epics/FE-04-feature-modules.md)  
**Status:** `[x]` Done

## Description
Implement and test consistent loading, empty, and error state patterns using shared components across all feature pages.

## Acceptance Criteria
- [x] A shared `<LoadingSpinner>` component renders during data fetch (`isLoading` state).
- [x] A shared `<EmptyState>` component renders when the data array is empty.
- [x] A shared `<ErrorMessage>` component renders when the query fails (`isError` state).
- [x] All three states are rendered correctly in the first feature page.
- [x] States are covered by component tests (see FE-05).
- [x] Shared state components are in `src/components/feedback/` (or equivalent shared folder).

## Technical Notes
- Use TanStack Query's `isLoading`, `isError`, `data` flags to conditionally render states.
- Do not use separate loading/error state in `useState` when TanStack Query already provides it.

## Definition of Done
All three states render correctly; shared components exist in the shared folder; tests pass.
