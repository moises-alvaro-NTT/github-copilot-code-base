# FE-01-05 — Set Up Path Aliases and Folder Structure

**Epic:** [FE-01 — Project Setup](../epics/FE-01-project-setup.md)  
**Status:** `[x]` Done

## Description
Create the agreed folder structure and configure TypeScript path aliases so features can import from `@/components`, `@/features`, `@/lib`, etc.

## Acceptance Criteria
- [x] Folders created: `src/app/`, `src/features/`, `src/components/`, `src/lib/`, `src/styles/`.
- [x] Path alias `@/` maps to `src/` in both `tsconfig.json` (paths) and Vite config (resolve.alias).
- [x] A sample import `import { cn } from '@/lib/utils'` resolves correctly.
- [x] Folder structure matches the layout in `copilot-instructions.md`.
- [x] `README.md` under `src/` (or root) briefly describes the folder purpose.

## Technical Notes
- Both `tsconfig.json` and `vite.config.ts` must configure aliases — TypeScript uses `paths`, Vite uses `resolve.alias`.
- Use `@` prefix (e.g., `@/features/`) for consistency.

## Definition of Done
All folders exist; aliases resolve correctly in IDE and in `npm run build`; no TS path errors.
