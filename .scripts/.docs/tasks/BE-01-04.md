# BE-01-04 — Configure Structured Logging with Correlation ID

**Epic:** [BE-01 — API Foundation](../epics/BE-01-api-foundation.md)  
**Status:** `[x]` Done

## Description
Configure Serilog (or the built-in `ILogger`) with structured output and ensure every log entry is enriched with a `correlation-id` derived from the incoming request.

## Acceptance Criteria
- [x] Structured logging outputs JSON in production and human-readable text in development.
- [x] Every request generates a unique `correlation-id` (check header `X-Correlation-Id` or generate one).
- [x] `correlation-id` is present in every log entry associated with that request.
- [x] `correlation-id` is returned to the caller in the response headers.
- [x] Log level is configurable via `appsettings.json` without recompiling.
- [x] No sensitive data (passwords, tokens) is logged.

## Technical Notes
- Use `Serilog.AspNetCore` with `RequestLogging` middleware for automatic request logs.
- Enrich with `CorrelationId` using a middleware that reads/generates the header and sets it on `HttpContext`.

## Definition of Done
Every log statement for a given request contains the same `correlation-id`; response header includes it.
