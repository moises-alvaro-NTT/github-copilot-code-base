# BE-04-04 — Create Thin `RenegotiationController` (No Logic)

**Epic:** [BE-04 — Renegotiation Feature](../epics/BE-04-renegotiation-feature.md)  
**Status:** `[x]` Done

## Description
Implement the HTTP controller that exposes the renegotiation endpoint.
The controller's only job is to receive the HTTP request, delegate to `IRenegotiationService`, and map the result to an HTTP response.

## Acceptance Criteria
- [x] `RenegotiationController` inherits `ControllerBase` and is annotated with `[ApiController]`.
- [x] `IRenegotiationService` is injected via constructor only.
- [x] Controller action contains **no** `if`, `switch`, `foreach`, business conditions, or persistence calls.
- [x] Controller action calls exactly one service method and maps the response.
- [x] Route is `POST /api/renegotiations`.
- [x] Response: `200 OK` with result DTO on success; `400 Bad Request` with ProblemDetails on validation error.
- [x] All public action methods have XML documentation comments.

## Technical Notes
- The controller must not call `new`, access repositories, or check entity state.
- Mapping of service result to HTTP status codes is the only allowed logic in the controller.

## Definition of Done
Controller compiles; code review confirms zero business logic in any action method.
