# BE-02-03 ‚Äî Implement Domain Rules and Validation

**Epic:** [BE-02 ‚Äî Domain Model](../epics/BE-02-domain-model.md)  
**Status:** `[x]` Done

## Description
Encode business invariants directly in entities and value objects using guard clauses.
Violations must fail fast with descriptive typed domain exceptions.

## Acceptance Criteria
- [x] Business rules are enforced in entity factory methods or constructors (not in controllers or services).
- [x] Invalid state throws a typed domain exception (e.g., `InvalidContractStateException`).
- [x] Domain exceptions inherit from a base `DomainException` class.
- [x] Guard clauses include a descriptive message for every branch.
- [x] No rule is duplicated ‚Äî each invariant is encoded exactly once.
- [x] All domain exception types have XML documentation.

## Technical Notes
- Consider a static `DomainExceptions` class with factory methods for consistent messages.
- Use `ArgumentException`/`InvalidOperationException` only for true programming errors, not business rules.

## Definition of Done
Invalid entity state throws the correct typed exception with a useful message; all rules covered by unit tests (see BE-02-04).
