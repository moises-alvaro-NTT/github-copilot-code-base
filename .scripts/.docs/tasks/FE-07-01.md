# FE-07-01 — Enforce Strict TypeScript Across All Modules

**Epic:** [FE-07 — Quality and Standards](../epics/FE-07-quality-standards.md)  
**Status:** `[x]` Done

## Description
Audit all TypeScript files and eliminate `any` types, implicit type widening, and unsafe casts. Ensure `strict: true` covers the entire source tree.

## Acceptance Criteria
- [x] `strict: true` in `tsconfig.json` with no exceptions.
- [x] Zero `any` types in production source files.
- [x] Zero `// @ts-ignore` or `// @ts-expect-error` comments without justification comment.
- [x] `noImplicitAny`, `strictNullChecks`, and `strictFunctionTypes` are all enabled (covered by `strict`).
- [x] `npm run build` exits 0 with no TypeScript errors.
- [x] ESLint `@typescript-eslint/no-explicit-any` rule is enabled as an error.

## Technical Notes
- Use `// @ts-expect-error` (not `@ts-ignore`) when suppression is unavoidable; add a comment explaining why.
- Zod schemas are the preferred way to type external data instead of type assertions.

## Definition of Done
Zero `any` types; `npm run build` exits 0; ESLint exits 0.
