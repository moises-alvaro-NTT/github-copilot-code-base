# FE-02-05 — Add clsx + tailwind-merge Utilities

**Epic:** [FE-02 — Core Libraries Integration](../epics/FE-02-core-libraries.md)  
**Status:** `[x]` Done

## Description
Install `clsx` and `tailwind-merge` and expose them through a shared `cn()` utility function used by all components for conditional class composition.

## Acceptance Criteria
- [x] `clsx` and `tailwind-merge` are installed.
- [x] A `cn()` utility is defined in `src/lib/utils.ts`.
- [x] `cn()` signature: `cn(...inputs: ClassValue[]): string`.
- [x] `cn()` correctly merges conflicting Tailwind classes (e.g., `cn('p-4', 'p-2')` → `'p-2'`).
- [x] All existing component class assignments use `cn()` (no raw template literals for conditional classes).
- [x] `cn()` is covered by at least 3 unit tests (merge conflict, conditional class, empty input).

## Technical Notes
- Implementation: `export function cn(...inputs: ClassValue[]) { return twMerge(clsx(inputs)); }`.
- Import `ClassValue` from `clsx`.

## Definition of Done
`cn()` utility exists in `lib/utils.ts`; all component class bindings use it; unit tests pass.
