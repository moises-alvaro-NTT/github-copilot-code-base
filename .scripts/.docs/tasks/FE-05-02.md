# FE-05-02 — Component Tests for Shared UI (Behavior + Accessibility)

**Epic:** [FE-05 — Testing](../epics/FE-05-testing.md)  
**Status:** `[x]` Done

## Description
Write component tests for all foundational shared UI components, covering all variants, interactive states, and accessibility requirements.

## Acceptance Criteria
- [x] `Button`: tests for each variant (primary, secondary, destructive), disabled state blocks click, keyboard Enter triggers onClick.
- [x] `Input`: error message renders with `aria-invalid`, label is associated via `htmlFor/id`.
- [x] `Alert`: each severity variant renders the correct icon/class.
- [x] `Card interactive`: keyboard Enter/Space triggers the action.
- [x] All assertions use accessible queries (`getByRole`, `getByLabelText`) over `getByTestId`.
- [x] `@testing-library/jest-dom` matchers used (`toBeDisabled`, `toHaveClass`, etc.).
- [x] axe-core (`jest-axe`) checks pass for each component.

## Technical Notes
- Install `jest-axe` for programmatic accessibility assertions.
- Prefer `userEvent` over `fireEvent` for simulating real user interactions.

## Definition of Done
All component tests pass; axe-core reports zero violations for all components.
