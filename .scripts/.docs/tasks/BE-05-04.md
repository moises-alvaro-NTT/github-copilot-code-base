# BE-05-04 — Integration Tests for Repositories

**Epic:** [BE-05 — Persistence](../epics/BE-05-persistence.md)  
**Status:** `[x]` Done

## Description
Write integration tests that exercise the repository methods against a real (or in-memory) database to verify CRUD operations and query correctness.

## Acceptance Criteria
- [x] Tests cover: Create entity → Read back → Update → Delete.
- [x] Read query tests verify that `AsNoTracking` entities are not change-tracked.
- [x] Each test starts with a clean database state (no data leakage between tests).
- [x] Tests use EF Core In-Memory provider or `Testcontainers` (PostgreSQL/SQL Server).
- [x] Test names follow `Method_Condition_ExpectedResult` in English.
- [x] Tests assert on returned DTO values — not on internal `DbContext` state.

## Technical Notes
- Prefer `Testcontainers` for closer production parity; EF In-Memory is acceptable for basic coverage.
- Wrap each test in a transaction or recreate the context between tests for isolation.

## Definition of Done
All CRUD integration tests pass; database is isolated between test runs.
