# BE-01-03 — Add Global Error Handling Middleware (ProblemDetails)

**Epic:** [BE-01 — API Foundation](../epics/BE-01-api-foundation.md)  
**Status:** `[x]` Done

## Description
Implement a global exception handling middleware that catches all unhandled exceptions and returns a structured `ProblemDetails` response (RFC 7807).
Stack traces must never be included in the response body in any environment.

## Acceptance Criteria
- [x] A `GlobalExceptionHandlerMiddleware` (or equivalent) is registered in `Program.cs`.
- [x] Unhandled exceptions return `500 Internal Server Error` with a `ProblemDetails` body.
- [x] Known domain exceptions map to the correct HTTP status code (e.g., `DomainNotFoundException` → 404).
- [x] Stack traces are never included in the response body.
- [x] Correlation ID is included in the response `ProblemDetails.Extensions`.
- [x] Error handling middleware integration test verifies: unknown exception → 500 ProblemDetails format.

## Technical Notes
- Use `app.UseExceptionHandler()` or a custom `IExceptionHandler` (ASP.NET 8+).
- Use `Microsoft.AspNetCore.Mvc.ProblemDetails` type for serialization consistency.

## Definition of Done
Any uncaught exception returns `ProblemDetails` JSON; no stack trace in response; correlation ID present.
