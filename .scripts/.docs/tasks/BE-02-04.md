# BE-02-04 — Unit Tests for Domain Rules

**Epic:** [BE-02 — Domain Model](../epics/BE-02-domain-model.md)  
**Status:** `[x]` Done

## Description
Write unit tests that verify every domain rule, invariant, and value object equality defined in BE-02-01 and BE-02-03.

## Acceptance Criteria
- [x] One test class per entity or value object under test.
- [x] Every public entity method has at least one passing and one failing test case.
- [x] All value object equality combinations are tested (equal, not-equal, same-reference).
- [x] All domain rule violations produce the expected typed exception.
- [x] Test names follow `Method_Condition_ExpectedResult` convention.
- [x] All test code is in English.
- [x] Tests are deterministic (no randomness, no date/time dependencies without injection).

## Technical Notes
- Use xUnit.
- No mocks needed — these are pure domain unit tests.
- Use `Assert.Throws<TException>()` for exception assertions.

## Definition of Done
All domain rule unit tests pass; no test depends on external state or infrastructure.
