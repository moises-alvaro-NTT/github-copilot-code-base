# BE-01-01 — Define Project Layer Structure

**Epic:** [BE-01 — API Foundation](../epics/BE-01-api-foundation.md)  
**Status:** `[x]` Done

## Description
Create the four .NET projects that form the Clean Architecture skeleton: `Domain`, `Application`, `Infrastructure`, and `Api`.
Set the project references so dependencies flow strictly inward. No build circular references are allowed.

## Acceptance Criteria
- [x] `src/backend/Domain/` project created targeting `net8.0`.
- [x] `src/backend/Application/` project references only `Domain`.
- [x] `src/backend/Infrastructure/` project references only `Application` (and therefore transitively `Domain`).
- [x] `src/backend/Api/` project references only `Application` and `Infrastructure` (for DI wiring).
- [x] Solution file includes all four projects.
- [x] `dotnet build` succeeds with zero errors.

## Technical Notes
- Use `dotnet new classlib` for Domain, Application, and Infrastructure.
- Use `dotnet new webapi` for Api.
- Enforce reference rules via `<ProjectReference>` entries — do not add circular refs.
