# FE-04-03 — Add First Feature Module (Route + Page + Data Hook)

**Epic:** [FE-04 — Feature Modules](../epics/FE-04-feature-modules.md)  
**Status:** `[x]` Done

## Description
Implement the first complete feature module end-to-end: route registration, a page component, and a TanStack Query data hook.

## Acceptance Criteria
- [x] Feature module is registered as a route in the app router.
- [x] Page component renders and is reachable via navigation.
- [x] A `useFeatureData` hook uses `useQuery` to fetch from the typed API client.
- [x] Page component uses the hook and renders the returned data.
- [x] Page component has no direct API calls — all data flows through the hook.
- [x] No business logic inside the page component.
- [x] TypeScript types flow from Zod schema → API function → hook → component props.

## Technical Notes
- Feature data hook: `export function useRenegotiations() { return useQuery({...}) }`.
- Keep the page component focused on rendering; delegate behavior to hooks and child components.

## Definition of Done
Feature route renders; data hook fetches and returns typed data; page displays it without errors.
