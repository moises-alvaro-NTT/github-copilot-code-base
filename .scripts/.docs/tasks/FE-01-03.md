# FE-01-03 — Add ESLint + Prettier + Strict TypeScript Config

**Epic:** [FE-01 — Project Setup](../epics/FE-01-project-setup.md)  
**Status:** `[x]` Done

## Description
Configure ESLint with React, TypeScript, and accessibility plugins. Configure Prettier for consistent code formatting. Ensure both run in CI.

## Acceptance Criteria
- [x] ESLint is configured with: `@typescript-eslint`, `eslint-plugin-react-hooks`, `eslint-plugin-jsx-a11y`.
- [x] `eslint-config-prettier` is added to avoid conflicts between ESLint and Prettier.
- [x] `npm run lint` reports zero errors on the initial clean project.
- [x] Prettier config (`.prettierrc`) is committed to source control.
- [x] A `lint-staged` + `husky` pre-commit hook runs lint and format on staged files.
- [x] `npm run format:check` validates formatting in CI.

## Technical Notes
- Use flat ESLint config (`eslint.config.js`) if on ESLint v9+.
- Enable `react-hooks/exhaustive-deps` rule as an error, not a warning.

## Definition of Done
`npm run lint` exits 0; `npm run format:check` exits 0; pre-commit hook blocks unformatted commits.
