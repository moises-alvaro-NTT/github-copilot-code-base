# FE-06-04 — Add Accessibility Notes per Documented Component

**Epic:** [FE-06 — Documentation and Playground](../epics/FE-06-documentation-playground.md)  
**Status:** `[x]` Done

## Description
Add accessibility notes to each component's story documentation describing keyboard interaction, ARIA roles, and focus behavior.

## Acceptance Criteria
- [x] Each foundational component story includes a written accessibility section.
- [x] Accessibility notes cover: keyboard interaction (Tab, Enter, Space), ARIA roles used, focus visible behavior.
- [x] Notes explain any accessibility trade-offs or known limitations.
- [x] Documentation uses Storybook MDX or `parameters.docs.description` for the notes.
- [x] All notes are in English.

## Technical Notes
- Use `parameters: { docs: { description: { component: '...' } } }` in story metadata for component-level notes.
- Or use `.mdx` files for richer documentation if needed.

## Definition of Done
Every foundational component story shows accessibility notes in the playground.
