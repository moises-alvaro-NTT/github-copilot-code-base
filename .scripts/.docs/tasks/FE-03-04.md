# FE-03-04 — Build Foundational Components (Button, Input, Card, Alert)

**Epic:** [FE-03 — Component Architecture](../epics/FE-03-component-architecture.md)  
**Status:** `[x]` Done

## Description
Implement the four foundational shared UI components using CVA, Tailwind design tokens, and the `cn()` utility.

## Acceptance Criteria
- [x] `Button`: variants `primary`, `secondary`, `destructive`; sizes `sm`, `md`, `lg`; `disabled` state visually and functionally.
- [x] `Input`: `label` prop, `error` state with message, `disabled` state; `id` linked to `label` for accessibility.
- [x] `Card`: `default` and `interactive` (clickable/hoverable) variants; accepts `children`.
- [x] `Alert`: severity variants `info`, `success`, `warning`, `error`; accepts `title` and `children`.
- [x] All components accept and spread a `className` prop for extension.
- [x] All component props have TypeScript types (no `any`).
- [x] All components render correctly in Storybook/Ladle stories (see FE-06-02).

## Technical Notes
- Build `Button` first as the CVA baseline; reuse the pattern for the others.
- Use `React.forwardRef` where the component wraps a native element (`input`, `button`).

## Definition of Done
All four components compile, render correctly in browser, and pass accessibility tests.
