# FE-03-03 — Implement Variant System (class-variance-authority)

**Epic:** [FE-03 — Component Architecture](../epics/FE-03-component-architecture.md)  
**Status:** `[x]` Done

## Description
Install `class-variance-authority` (CVA) and integrate it as the standard variant system for all UI components.

## Acceptance Criteria
- [x] `class-variance-authority` is installed.
- [x] A sample `buttonVariants` is defined using `cva()` with at least `variant` and `size` variant keys.
- [x] Component props are typed from the CVA variant config using `VariantProps<typeof componentVariants>`.
- [x] `cn()` from `lib/utils` is used alongside CVA for additional class merging.
- [x] A test verifies that the correct CSS classes are applied for each variant combination.
- [x] No hardcoded conditional class logic (`x ? 'class-a' : 'class-b'`) outside of CVA definitions.

## Technical Notes
- Use `cva('base-classes', { variants: { variant: {...}, size: {...} }, defaultVariants: {...} })`.
- CVA + `cn()` is the canonical pattern for all conditional styling.

## Definition of Done
CVA integrated; sample component uses variant config; conditional class logic is eliminated outside CVA.
