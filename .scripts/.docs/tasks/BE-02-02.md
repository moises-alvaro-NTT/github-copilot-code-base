# BE-02-02 — Define Domain Events

**Epic:** [BE-02 — Domain Model](../epics/BE-02-domain-model.md)  
**Status:** `[x]` Done

## Description
Define domain events that entities raise when significant state changes occur.
Events are value objects and contain only the data required by consumers — no behavior.

## Acceptance Criteria
- [x] A base `IDomainEvent` interface or `DomainEvent` base record is defined.
- [x] Each significant entity state change raises the corresponding domain event.
- [x] Domain events are collected in the aggregate root and accessible via `DomainEvents` property.
- [x] Domain events do not contain references to repositories, services, or infrastructure types.
- [x] Events are named in past tense (e.g., `ContractRenegotiatedEvent`).
- [x] All event types have XML documentation comments.

## Technical Notes
- Aggregate root accumulates events in a private `List<IDomainEvent>` and exposes `ReadOnlyCollection<IDomainEvent> DomainEvents`.
- Events are dispatched by the Infrastructure layer after persistence.

## Definition of Done
At least one domain event is defined and raised inside an aggregate root method; domain layer compiles clean.
