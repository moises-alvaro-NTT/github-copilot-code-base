# BE-05-01 — Configure EF Core DbContext

**Epic:** [BE-05 — Persistence](../epics/BE-05-persistence.md)  
**Status:** `[x]` Done

## Description
Set up the EF Core `DbContext` in `Infrastructure`, configure entity mappings, and register it in DI with the correct connection string handling.

## Acceptance Criteria
- [x] `AppDbContext` inherits `DbContext` and is in `Infrastructure/Persistence/`.
- [x] Entity mappings are configured in `IEntityTypeConfiguration<T>` classes (not via Data Annotations on domain entities).
- [x] Connection string is read from `IConfiguration` — never hardcoded.
- [x] `DbContext` is registered with `AddDbContext<AppDbContext>()` inside `AddInfrastructureServices()`.
- [x] `Domain` entities have zero EF Core attributes (`[Key]`, `[Table]`, etc.).
- [x] `dotnet ef migrations add Initial` succeeds on a clean database.

## Technical Notes
- Use `Fluent API` exclusively for EF mappings; this keeps Domain free of infrastructure coupling.
- Use `appsettings.json` / environment variables for the connection string.

## Definition of Done
`AppDbContext` compiles; migrations apply cleanly; Domain entities have no EF attributes.
