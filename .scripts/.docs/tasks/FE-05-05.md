# FE-05-05 — Enforce `ComponentOrHook_Condition_ExpectedResult` Naming

**Epic:** [FE-05 — Testing](../epics/FE-05-testing.md)  
**Status:** `[x]` Done

## Description
Audit all existing frontend test names and enforce the `ComponentOrHook_Condition_ExpectedResult` convention. Configure ESLint to warn on deviations.

## Acceptance Criteria
- [x] All existing test names comply with `ComponentOrHook_Condition_ExpectedResult`.
- [x] All test names are in English.
- [x] An ESLint rule or comment enforces the convention (e.g., `eslint-plugin-vitest` `consistent-test-it` + naming pattern).
- [x] CI fails if a test name does not match the pattern.
- [x] A doc/comment in the test folder explains the naming convention.

## Technical Notes
- `eslint-plugin-vitest` provides `vitest/consistent-test-it` and custom naming rules.
- Consider adding a regex-based naming rule or a Jest/Vitest `--reporter` that flags non-compliant names.

## Definition of Done
All test names comply; ESLint rule is configured; CI enforces it.
