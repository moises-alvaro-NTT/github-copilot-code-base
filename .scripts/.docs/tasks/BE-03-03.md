# BE-03-03 — Add FluentValidation for Request DTOs

**Epic:** [BE-03 — Application Use Cases](../epics/BE-03-application-use-cases.md)  
**Status:** `[x]` Done

## Description
Add FluentValidation validators for all incoming request DTOs in the Application layer.
Validation must execute before any handler logic and return structured error details.

## Acceptance Criteria
- [x] `FluentValidation.AspNetCore` is installed in the `Application` project.
- [x] One validator class exists per request DTO (`CreateContractRequestValidator`, etc.).
- [x] All required fields, length constraints, and business-format rules are captured in validators.
- [x] Validation failures return `400 Bad Request` with `ProblemDetails` listing field errors.
- [x] Validators are registered automatically via `AddValidatorsFromAssembly`.
- [x] Validator unit tests cover: valid input passes, each invalid field fails with the correct message.

## Technical Notes
- Register validators in `AddApplicationServices()` extension.
- Use `AbstractValidator<TRequest>` base class.
- Consider a pipeline behavior (or middleware) that runs validation before the handler.

## Definition of Done
Invalid DTO inputs are rejected at the validation stage before the handler executes; all validators are covered by unit tests.
