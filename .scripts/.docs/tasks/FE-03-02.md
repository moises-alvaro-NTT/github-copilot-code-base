# FE-03-02 — Create Base Design Tokens (Colors, Spacing, Typography)

**Epic:** [FE-03 — Component Architecture](../epics/FE-03-component-architecture.md)  
**Status:** `[x]` Done

## Description
Define all design tokens (brand colors, neutral palette, spacing scale, font sizes, border radii) in `tailwind.config.ts` as `theme.extend` values.

## Acceptance Criteria
- [x] Brand colors are defined under `theme.extend.colors` (e.g., `primary`, `secondary`, `destructive`, `muted`).
- [x] Font families are defined under `theme.extend.fontFamily`.
- [x] Border radius scale includes at least `sm`, `md`, `lg`, `full`.
- [x] All token names are semantic (e.g., `text-primary` not `text-blue-600`).
- [x] Design token file (e.g., `src/styles/tokens.ts`) exports the tokens imported by `tailwind.config.ts`.
- [x] A visual sample page (dev-only) renders all tokens for visual verification.

## Technical Notes
- Avoid hardcoding `text-blue-600` in components — always use semantic token names.
- Token naming should map to shadcn/ui conventions if that library will be used later.

## Definition of Done
Tailwind config extended with semantic tokens; components use semantic class names; tokens visually verified.
