# FE-05-03 — Integration Tests for Critical Async Flows

**Epic:** [FE-05 — Testing](../epics/FE-05-testing.md)  
**Status:** `[x]` Done

## Description
Write integration tests that render a feature page component and mock the network layer to verify loading → data → error state transitions.

## Acceptance Criteria
- [x] Test: `FeaturePage_OnMount_ShowsLoadingSpinner`
- [x] Test: `FeaturePage_WhenDataLoads_RendersListItems`
- [x] Test: `FeaturePage_WhenApiReturnsEmpty_ShowsEmptyState`
- [x] Test: `FeaturePage_WhenApiFails_ShowsErrorMessage`
- [x] Network is mocked at the `fetch` boundary using `msw` (Mock Service Worker) or `vi.spyOn(global, 'fetch')`.
- [x] Tests do not mock React Query internals — they mock the network.
- [x] Test names follow `ComponentOrHook_Condition_ExpectedResult` convention.

## Technical Notes
- Prefer MSW for integration tests — it intercepts at the network layer, not the function level.
- Wrap render with `QueryClientProvider` and a fresh `QueryClient` per test.

## Definition of Done
All four async flow tests pass; no real HTTP calls made during test runs.
