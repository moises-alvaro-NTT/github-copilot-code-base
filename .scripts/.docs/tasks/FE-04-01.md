# FE-04-01 — Define Feature-First Folder Structure

**Epic:** [FE-04 — Feature Modules](../epics/FE-04-feature-modules.md)  
**Status:** `[x]` Done

## Description
Establish the folder structure convention for feature modules and document it so all future features are organized consistently.

## Acceptance Criteria
- [x] A sample feature folder exists: `src/features/<feature-name>/`.
- [x] Sub-structure inside each feature: `components/`, `hooks/`, `pages/`, `schemas/`, `types/`.
- [x] Feature modules export only what is needed externally via a barrel `index.ts`.
- [x] Feature-specific components are never imported directly by other features (go through the barrel).
- [x] A `README.md` in `src/features/` documents the structure and conventions.

## Technical Notes
- Avoid importing from `../../features/other-feature/` directly; use barrel exports.
- Shared code that multiple features need goes in `src/components/` or `src/lib/`.

## Definition of Done
One sample feature folder exists with the agreed structure; cross-feature imports go through barrel.
