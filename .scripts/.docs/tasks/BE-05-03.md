# BE-05-03 — Add Migrations

**Epic:** [BE-05 — Persistence](../epics/BE-05-persistence.md)  
**Status:** `[x]` Done

## Description
Create the initial EF Core migration that represents the first database schema and verify it applies cleanly to a fresh database.

## Acceptance Criteria
- [x] `dotnet ef migrations add Initial` generates migration files without errors.
- [x] `dotnet ef database update` applies the migration to a local/test database without errors.
- [x] Migration is idempotent: running `update` twice does not throw.
- [x] Migration files are committed to source control.
- [x] `__EFMigrationsHistory` table is the only schema management artifact.
- [x] Snapshot file is generated and up-to-date with the model.

## Technical Notes
- Add `Microsoft.EntityFrameworkCore.Design` to the `Api` or a dedicated `Migrations` project.
- Use `--project Infrastructure --startup-project Api` flags in migration commands.

## Definition of Done
Migration files exist in source control; `dotnet ef database update` applies cleanly on a fresh database.
