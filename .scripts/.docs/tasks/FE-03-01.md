# FE-03-01 — Define Shared UI Component Folder

**Epic:** [FE-03 — Component Architecture](../epics/FE-03-component-architecture.md)  
**Status:** `[x]` Done

## Description
Create the `src/components/ui/` folder structure and establish the conventions for how shared components are organized, exported, and documented.

## Acceptance Criteria
- [x] `src/components/ui/` folder exists with an `index.ts` barrel export.
- [x] A `README.md` inside `components/` describes the purpose of each subfolder.
- [x] Components use named exports (not default exports).
- [x] Each component file is co-located with its test file (`Button.tsx` + `Button.test.tsx`).
- [x] Component files use PascalCase naming.
- [x] No business logic exists in any file under `components/ui/`.

## Technical Notes
- Structure: `components/ui/Button/Button.tsx`, `Button.test.tsx`, `index.ts`.
- Or flat: `components/ui/Button.tsx`, `Button.test.tsx` — pick one and be consistent.

## Definition of Done
Folder structure exists; barrel export works; conventions documented.
