# FE-06-02 — Add Stories for Foundational Components (All States)

**Epic:** [FE-06 — Documentation and Playground](../epics/FE-06-documentation-playground.md)  
**Status:** `[x]` Done

## Description
Write Storybook/Ladle stories for all four foundational components, covering every variant and interactive state.

## Acceptance Criteria
- [x] `Button.stories.tsx`: stories for primary, secondary, destructive, and disabled.
- [x] `Input.stories.tsx`: stories for default, with error message, and disabled.
- [x] `Card.stories.tsx`: stories for default and interactive variant.
- [x] `Alert.stories.tsx`: stories for info, success, warning, and error severity.
- [x] Each story includes a meaningful name and uses `args` for interactive controls.
- [x] All stories use realistic content (no "Lorem ipsum" or placeholder-only text).
- [x] Stories are type-safe (`Meta<typeof Component>` and `StoryObj<typeof Component>`).

## Technical Notes
- Use CSF3 format (Component Story Format 3) for all stories.
- Use Storybook `argTypes` to expose variant controls in the UI.

## Definition of Done
All four component story files exist; each story renders in the playground with correct styles.
