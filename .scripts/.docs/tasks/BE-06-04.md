# BE-06-04 — Configure Nullable and XML Doc Warnings in csproj

**Epic:** [BE-06 — Quality and Standards](../epics/BE-06-quality-standards.md)  
**Status:** `[x]` Done

## Description
Enable nullable reference types and XML documentation generation in all production project files, ensuring no public member is left undocumented.

## Acceptance Criteria
- [x] `<Nullable>enable</Nullable>` is present in all production `.csproj` files.
- [x] `<GenerateDocumentationFile>true</GenerateDocumentationFile>` is set in all production `.csproj` files.
- [x] `<TreatWarningsAsErrors>true</TreatWarningsAsErrors>` is set (or equivalent warning-level enforcement).
- [x] `dotnet build` exits 0 with no nullable or documentation warnings.
- [x] Test projects do not require full XML doc coverage (suppress `CS1591` in test projects).

## Technical Notes
- Use `<NoWarn>CS1591</NoWarn>` in test project `.csproj` files to suppress missing XML doc warnings.
- Add `<WarningsAsErrors>CS8600;CS8602;CS8603</WarningsAsErrors>` as a minimum nullable enforcement set if full `TreatWarningsAsErrors` is too aggressive initially.

## Definition of Done
`dotnet build` exits 0 with nullable and XML doc checks enabled in all production projects.
