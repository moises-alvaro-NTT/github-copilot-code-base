# BE-03-02 — Implement Use Case Handlers

**Epic:** [BE-03 — Application Use Cases](../epics/BE-03-application-use-cases.md)  
**Status:** `[x]` Done

## Description
Implement the use case orchestration classes in `Application/UseCases/`.
Each handler receives its dependencies through constructor injection, delegates persistence to repository interfaces, and delegates domain logic to entities.

## Acceptance Criteria
- [x] One handler class per use case (single responsibility).
- [x] Handlers receive interfaces only through constructor injection (no `new` for services).
- [x] Handlers do not directly reference `DbContext` or any EF Core type.
- [x] Handlers call domain entity methods for business logic — no business logic inline.
- [x] All handler methods are `async` and propagate `CancellationToken`.
- [x] All public handler methods have XML documentation.
- [x] Handlers return a typed result (`Result<T>` or `Response DTO`) — no raw exceptions escaping to the controller.

## Technical Notes
- Consider a lightweight `Result<T>` type for success/failure without exception flow.
- Keep handler class focused: one public `ExecuteAsync` or `HandleAsync` method.

## Definition of Done
Each planned use case has a handler that compiles, injects interfaces correctly, and has no direct persistence dependencies.
