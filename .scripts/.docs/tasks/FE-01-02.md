# FE-01-02 — Configure Tailwind CSS with Design Tokens

**Epic:** [FE-01 — Project Setup](../epics/FE-01-project-setup.md)  
**Status:** `[x]` Done

## Description
Install and configure Tailwind CSS with a custom `tailwind.config.ts` that extends the default theme with the project's design tokens (colors, spacing, typography, border radius).

## Acceptance Criteria
- [x] Tailwind CSS and PostCSS are installed and configured.
- [x] `tailwind.config.ts` extends theme with at least: custom color palette, font family, and spacing scale.
- [x] Global CSS file imports Tailwind base, components, and utilities directives.
- [x] A sample component using a custom color class renders correctly.
- [x] `npm run build` includes only used Tailwind classes (PurgeCSS/content config is set).

## Technical Notes
- Add `content: ['./index.html', './src/**/*.{ts,tsx}']` to the Tailwind config.
- Store token definitions as a separate `tokens.ts` file imported in `tailwind.config.ts`.

## Definition of Done
Tailwind works end-to-end; custom design tokens apply in browser; unused CSS is purged in production build.
