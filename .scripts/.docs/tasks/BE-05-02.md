# BE-05-02 — Implement Repositories with `AsNoTracking` Read Pattern

**Epic:** [BE-05 — Persistence](../epics/BE-05-persistence.md)  
**Status:** `[x]` Done

## Description
Implement the repository classes in `Infrastructure/Persistence/Repositories/` that satisfy the interfaces defined in `Application/Contracts/`.
All read queries must use `AsNoTracking()` to avoid unintentional entity tracking.

## Acceptance Criteria
- [x] Each repository class implements its corresponding interface from `Application/Contracts/`.
- [x] All read-only queries use `.AsNoTracking()`.
- [x] `ToListAsync()` / `FirstOrDefaultAsync()` are called only at the final materialization point (no intermediate materialization).
- [x] Write operations (Add, Update, Remove) call `SaveChangesAsync()` after the change.
- [x] Repositories do not expose `IQueryable` outside the Infrastructure boundary.
- [x] All public repository methods have XML documentation.

## Technical Notes
- Pattern: `_context.Set<Entity>().AsNoTracking().Where(...).ToListAsync(ct)`.
- For writes: load entity with tracking, mutate via entity method, then `SaveChangesAsync()`.

## Definition of Done
All repository methods compile; read queries include `AsNoTracking()`; integration tests (BE-05-04) pass.
