# BE-04-01 — Define `IRenegotiationService` Interface in Application

**Epic:** [BE-04 — Renegotiation Feature](../epics/BE-04-renegotiation-feature.md)  
**Status:** `[x]` Done

## Description
Define the `IRenegotiationService` interface in `Application/Contracts/` that describes all renegotiation operations.
The controller will only know about this interface — never about the concrete implementation.

## Acceptance Criteria
- [x] `IRenegotiationService` is defined in `Application/Contracts/`.
- [x] The interface includes at minimum: `RenegotiateAsync(RenegotiationRequest request, CancellationToken ct)`.
- [x] `RenegotiationRequest` DTO and `RenegotiationResult` response type are defined in Application.
- [x] All method signatures use `async Task<T>` with `CancellationToken` parameter.
- [x] All interface members and DTO properties have XML documentation comments.
- [x] The interface contains zero references to Infrastructure or EF Core.

## Technical Notes
- Place DTO types in `Application/Contracts/Dtos/` or co-locate with the interface.
- Use `RenegotiationResult` as a typed result — avoid `object` return types.

## Definition of Done
Interface and DTOs compile in Application; no reference to Infrastructure exists.
