# BE-01-02 — Configure DI Container and Project References

**Epic:** [BE-01 — API Foundation](../epics/BE-01-api-foundation.md)  
**Status:** `[x]` Done

## Description
Wire up the dependency injection container in `Program.cs` using extension methods.
Each layer exposes a `AddXxxServices(this IServiceCollection)` extension to keep `Program.cs` clean and single-responsibility.

## Acceptance Criteria
- [x] `Application` exposes `AddApplicationServices()` extension method.
- [x] `Infrastructure` exposes `AddInfrastructureServices()` extension method.
- [x] `Program.cs` calls both extension methods — no service registered directly in `Program.cs`.
- [x] No service is instantiated with `new` outside of DI.
- [x] All registrations have explicit lifetimes (Scoped/Transient/Singleton).
- [x] `dotnet run` starts the API without runtime DI exceptions.

## Technical Notes
- Put extension methods in `DependencyInjection.cs` inside each project.
- Use `builder.Services.AddApplicationServices()` and `builder.Services.AddInfrastructureServices()` pattern.

## Definition of Done
API starts, all services resolve correctly, no `new ServiceName()` anywhere in production code.
