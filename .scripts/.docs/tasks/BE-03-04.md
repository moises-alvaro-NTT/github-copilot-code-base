# BE-03-04 — Unit Tests for Use Cases

**Epic:** [BE-03 — Application Use Cases](../epics/BE-03-application-use-cases.md)  
**Status:** `[x]` Done

## Description
Write unit tests for every use case handler, covering the success path, validation failure path, and domain error path.
Dependencies are mocked at the interface boundary.

## Acceptance Criteria
- [x] Every handler has at least one test for the success path.
- [x] Every handler has at least one test for validation failure.
- [x] Every handler has at least one test for a domain error (e.g., entity not found, rule violation).
- [x] Mocks target only the interface (not the concrete implementation).
- [x] Tests do not start a real database or real HTTP server.
- [x] Test names follow `Method_Condition_ExpectedResult`.
- [x] All test code and comments are in English.
- [x] Tests use Moq or NSubstitute for mocking.

## Technical Notes
- Test the handler in isolation: mock all constructor-injected interfaces.
- Assert on the returned `Result<T>` value, not on caught exceptions for expected failures.

## Definition of Done
All handler unit tests pass; no test has external dependencies.
