# BE-04-03 — Register `IRenegotiationService` in DI

**Epic:** [BE-04 — Renegotiation Feature](../epics/BE-04-renegotiation-feature.md)  
**Status:** `[x]` Done

## Description
Register `RenegotiationService` as the implementation of `IRenegotiationService` in the DI container with the correct lifetime.

## Acceptance Criteria
- [x] `services.AddScoped<IRenegotiationService, RenegotiationService>()` (or equivalent) is present in `AddInfrastructureServices()`.
- [x] Registration is in the `Infrastructure` DI extension, not in `Program.cs` directly.
- [x] Lifetime is `Scoped` (per-request) unless there is a documented reason for another choice.
- [x] The API starts without runtime DI exceptions related to `IRenegotiationService`.
- [x] A smoke test confirms the controller receives a non-null `IRenegotiationService` instance.

## Technical Notes
- Validate lifetime: `RenegotiationService` likely depends on `DbContext` which is Scoped; a Singleton service would cause a captive dependency issue.

## Definition of Done
`IRenegotiationService` resolves correctly from DI; no manual `new RenegotiationService()` anywhere.
