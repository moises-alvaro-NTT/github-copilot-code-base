# FE-07-03 — Audit and Fix Accessibility on Interactive Components

**Epic:** [FE-07 — Quality and Standards](../epics/FE-07-quality-standards.md)  
**Status:** `[x]` Done

## Description
Run a full accessibility audit on all interactive components using axe-core (automated) and keyboard-only manual testing. Fix all critical and serious violations.

## Acceptance Criteria
- [x] Automated axe-core tests pass for all interactive components (no critical/serious violations).
- [x] Manual keyboard-only test confirms all interactive elements are reachable and operable.
- [x] All form inputs have associated labels.
- [x] Color contrast meets WCAG 2.1 AA minimum (4.5:1 for normal text, 3:1 for large text).
- [x] No `outline: none` without a visible replacement focus style.
- [x] All images have meaningful `alt` text or `alt=""` for decorative images.
- [x] Live regions (`aria-live`) are used for dynamic content updates (errors, notifications).

## Technical Notes
- Use `@axe-core/react` in development mode for live accessibility feedback.
- Use browser DevTools accessibility panel + keyboard-only walkthrough for manual testing.

## Definition of Done
Zero axe-core critical/serious violations; manual keyboard navigation reaches all interactive elements.
