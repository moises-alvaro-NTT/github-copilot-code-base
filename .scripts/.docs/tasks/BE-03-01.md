# BE-03-01 — Define Application Contracts (Interfaces) for Each Capability

**Epic:** [BE-03 — Application Use Cases](../epics/BE-03-application-use-cases.md)  
**Status:** `[x]` Done

## Description
Define all service and repository interfaces in `Application/Contracts/`.
These interfaces are the dependency inversion boundary — Application depends on the contracts, not the implementations.

## Acceptance Criteria
- [x] All business capability interfaces are in `Application/Contracts/` (not in Infrastructure or Api).
- [x] Repository interfaces are defined here (e.g., `IContractRepository`).
- [x] Service interfaces are defined here (e.g., `IEmailNotificationService`).
- [x] Interfaces contain only the methods needed by use cases — no infrastructure leakage.
- [x] All interface methods are `async Task<T>` and accept `CancellationToken`.
- [x] All interfaces have XML documentation comments.

## Technical Notes
- Naming: prefix with `I` followed by the role name (`IContractRepository`, not `IRepository<Contract>`).
- Avoid generic repository patterns — prefer specific, intention-revealing interfaces.

## Definition of Done
All planned contracts are defined in `Application/Contracts/`; Domain project has zero references to Application.
