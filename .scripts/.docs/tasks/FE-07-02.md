# FE-07-02 — Validate All API Boundaries with Zod

**Epic:** [FE-07 — Quality and Standards](../epics/FE-07-quality-standards.md)  
**Status:** `[x]` Done

## Description
Audit the API client layer and ensure every API response is validated with a Zod schema before the data flows into the app.

## Acceptance Criteria
- [x] Every API client function calls `schema.parse(data)` or `schema.safeParse(data)` on the response.
- [x] No API response is typed with a TypeScript `as` cast without schema validation.
- [x] Zod schemas are in `src/lib/schemas/` and co-located with the API function or imported from a shared schema file.
- [x] A test verifies that an unexpected API response shape triggers a Zod validation error.
- [x] API client functions return typed domain objects, not raw `Response` or `any`.

## Technical Notes
- Use `schema.safeParse()` when you want to handle parse errors gracefully rather than throwing.
- Commit Zod schemas as the authoritative "API contract" documentation for each endpoint.

## Definition of Done
All API functions validate responses with Zod; no unvalidated `as` casts on API data; test confirms schema rejection.
