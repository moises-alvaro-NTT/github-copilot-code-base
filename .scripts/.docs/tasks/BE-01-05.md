# BE-01-05 — Add Health Check Endpoint

**Epic:** [BE-01 — API Foundation](../epics/BE-01-api-foundation.md)  
**Status:** `[x]` Done

## Description
Register the ASP.NET Core health checks and expose a `GET /health` endpoint that reflects live/readiness state of the service.

## Acceptance Criteria
- [x] `GET /health` returns `200 OK` with body `{"status":"Healthy"}` when all checks pass.
- [x] `GET /health` returns `503 Service Unavailable` when any check fails.
- [x] At least one tagged check exists (e.g., database connectivity).
- [x] Health endpoint is excluded from authentication requirements.
- [x] Health endpoint does not expose sensitive infrastructure details in the response body.

## Technical Notes
- Use `services.AddHealthChecks()` and `app.MapHealthChecks("/health")`.
- For database health, use the EF Core health check package or a raw connection test.

## Definition of Done
`curl http://localhost:<port>/health` returns `200 Healthy` in a running dev environment.
