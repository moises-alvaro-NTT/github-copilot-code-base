# FE-04-02 — Implement Typed API Client Layer

**Epic:** [FE-04 — Feature Modules](../epics/FE-04-feature-modules.md)  
**Status:** `[x]` Done

## Description
Create a typed API client in `src/lib/api/` that wraps `fetch` (or Axios) with Zod response validation at the boundary.

## Acceptance Criteria
- [x] API client functions are in `src/lib/api/` organized by resource (e.g., `renegotiations.api.ts`).
- [x] Every API function validates the response with a Zod schema before returning.
- [x] API functions are `async` and return typed results (no `any` or `unknown` unvalidated).
- [x] A shared `apiClient` helper handles base URL, authorization headers, and error mapping.
- [x] HTTP 4xx/5xx responses are mapped to typed error objects (not raw `Response` objects).
- [x] `Authorization` header is injected without exposing raw tokens in component code.

## Technical Notes
- Use `schema.parse(data)` at the response boundary — if the API returns unexpected shape, it throws immediately.
- Store the API base URL in an environment variable (`VITE_API_BASE_URL`).

## Definition of Done
One API function calls the backend, validates with Zod, and returns a typed result; no `any` in API layer.
