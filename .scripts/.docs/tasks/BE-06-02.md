# BE-06-02 — Enforce Naming Standard: `Method_Condition_ExpectedResult`

**Epic:** [BE-06 — Quality and Standards](../epics/BE-06-quality-standards.md)  
**Status:** `[x]` Done

## Description
Audit all existing test methods and enforce the `Method_Condition_ExpectedResult` naming convention.
Add an EditorConfig or Roslyn analyzer rule to catch violations automatically.

## Acceptance Criteria
- [x] All existing test method names follow `Method_Condition_ExpectedResult`.
- [x] All test names are in English.
- [x] `AAA` sections are labeled (`// Arrange`, `// Act`, `// Assert`) when section blocks are explicitly commented.
- [x] A `.editorconfig` entry or Roslyn analyzer is configured to warn on test methods that do not match the convention.
- [x] CI build fails if the naming convention is violated.

## Technical Notes
- Use `xunit.analyzers` NuGet package for xUnit-specific rules.
- For the naming rule, consider `dotnet_naming_rule` in `.editorconfig` or a custom Roslyn analyzer.

## Definition of Done
All test names comply; CI reports a warning or error on non-compliant new test names.
