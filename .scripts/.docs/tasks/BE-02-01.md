# BE-02-01 — Define Core Entities and Value Objects

**Epic:** [BE-02 — Domain Model](../epics/BE-02-domain-model.md)  
**Status:** `[x]` Done

## Description
Model the core business entities and value objects in the `Domain` project.
Entities protect their invariants via private setters and expose behavior through methods. Value objects are immutable with value-based equality.

## Acceptance Criteria
- [x] At least one aggregate root entity is defined with private setters.
- [x] Entity IDs are typed (e.g., `record ContractId(Guid Value)`) not raw `Guid`.
- [x] Value objects override `Equals` and `GetHashCode` for value-based equality.
- [x] Value objects are immutable (`readonly` fields or `init`-only properties).
- [x] No entity or value object depends on `Infrastructure`, `Application`, or any ORM attribute.
- [x] All public types have XML documentation comments.

## Technical Notes
- Prefer `record` types for value objects.
- Inherit from a base `Entity<TId>` class for common identity handling.

## Definition of Done
Entities and value objects compile, are free of framework dependencies, and all public members are documented.
